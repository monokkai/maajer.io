# Project Structure & Tooling Alignment Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Restructure the portfolio site to a `src/app` layout with a component-folder convention, a shared `types.ts`, and a `@/*` path alias — mirroring `luukasmakila/lukemadeit.io`'s organization — and add Prettier + Node version pinning, without touching dependency versions or inventing content pages.

**Architecture:** Move the existing `app/` directory under a new `src/` directory (Next.js auto-detects `src/app`), update the TypeScript path alias to point at `src`, add a minimal `Navbar` component under `src/app/components/Navbar/` wired into the root layout, add an empty `src/app/types.ts` for future shared types, then layer on Prettier (with `eslint-config-prettier` to avoid rule conflicts) and a `.nvmrc`.

**Tech Stack:** Next.js 16 (App Router), React 19, TypeScript 5, Tailwind CSS v4, ESLint 9 (flat config), npm.

## Global Constraints

- Keep all current dependency versions as-is (Next 16.2.10, React 19.2.4, Tailwind v4, TypeScript ^5) — no downgrades to match the reference project.
- Stay on npm (no package manager change, no yarn.lock).
- Path alias must be `@/*` → `./src/*`.
- No `tailwind.config.ts` — Tailwind v4's CSS-based config via `postcss.config.mjs` already covers this.
- No blog/gear/products pages or content-specific components — skeleton only.
- No Husky/lint-staged, no `.editorconfig`.

### Task 1: Move `app/` to `src/app/` and update the path alias

**Files:**

- Move: `app/` → `src/app/` (via `git mv`, preserves history for `favicon.ico`, `globals.css`, `layout.tsx`, `page.tsx`)

  ...nextTs,
